## Pregunta 30 (Páginas 64-66)

**Contexto**

DRAG DROP -
Eres desarrollador en una empresa que proporciona un servicio de gestión de reservas en la industria turística. Estás implementando Azure Search para las agencias de viajes que aparecen en la solución de tu empresa.
Ya creaste el índice en Azure Search. Ahora debes usar el SDK de Azure Search .NET para importar los datos relevantes al servicio de Azure Search.
¿Qué tres acciones debes realizar en secuencia? Para responder, mueve las acciones apropiadas de la lista de acciones de la izquierda a la derecha y ordénalas en el orden correcto.

El área de respuesta muestra una lista de posibles pasos (cada uno en un recuadro) y tres espacios vacíos a la derecha donde se deben colocar en orden. Los pasos listados son:
- "Create a DataSource instance and set its Container property to the DataContainer."
- "Create an IndexBatch that contains the documents which must be added."
- "Set the DataSources property of the SearchServiceClient."
- "Create a SearchIndexClient object to connect to the search index."
- "Call the Documents.Index method of the SearchIndexClient and pass the IndexBatch."
- "Call the Documents.Suggest method of the SearchIndexClient and pass the DataSource."
Debes elegir y ordenar tres de estas acciones.

**Opciones**

- **A)** Crear un objeto SearchIndexClient para conectarse al índice de búsqueda; crear un IndexBatch que contenga los documentos que deben agregarse; llamar al método Documents.Index de SearchIndexClient y pasar el IndexBatch.
- **B)** Crear una instancia de DataSource y establecer su propiedad Container en DataContainer; establecer la propiedad DataSources de SearchServiceClient; llamar al método Documents.Suggest de SearchIndexClient y pasar el DataSource.
- **C)** Crear una instancia de DataSource y establecer su propiedad Container en DataContainer; crear un IndexBatch que contenga los documentos que deben agregarse; llamar al método Documents.Suggest de SearchIndexClient y pasar el DataSource.
- **D)** Establecer la propiedad DataSources de SearchServiceClient; crear un objeto SearchIndexClient para conectarse al índice de búsqueda; llamar al método Documents.Index de SearchIndexClient y pasar el IndexBatch.

**Respuesta Correcta**

A: Crear un objeto SearchIndexClient para conectarse al índice de búsqueda; crear un IndexBatch que contenga los documentos que deben agregarse; llamar al método Documents.Index de SearchIndexClient y pasar el IndexBatch.

**Explicación**

El objetivo es cargar documentos en un índice ya creado utilizando el SDK de Azure Search para .NET. Según la guía oficial, primero se necesita un objeto SearchIndexClient que represente el índice de búsqueda y proporcione acceso a sus documentos. Una vez que se tiene el cliente, se construye un IndexBatch que contiene la colección de documentos a cargar (por ejemplo, una lista de objetos "Hotel"). Finalmente, se llama al método Documents.Index del SearchIndexClient pasando el IndexBatch, lo que envía la operación de indexación al servicio. Las acciones relacionadas con DataSource y la propiedad DataSources se usan cuando se configura una indexación basada en orígenes de datos (indexers), no para la carga directa de documentos. El método Documents.Suggest se emplea para sugerencias de búsqueda, no para importar datos. Por ello, la secuencia correcta es: 1) Crear un objeto SearchIndexClient; 2) Crear un IndexBatch con los documentos; 3) Llamar a Documents.Index con ese IndexBatch.
